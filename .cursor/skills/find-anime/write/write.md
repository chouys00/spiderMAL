## 寫入動畫清單到 MCP filesystem

### 專案位置

`d:\練習\spiderMAL`

### 功能說明

從專案根目錄的 `anime-cache.json` 讀取最近一次「找動畫」產生的暫存結果，將其格式化後寫入 `D:/MCPfilesystem/動畫.md`。

**完整流程：**

1. 使用者執行「找動畫」→ 產生 `anime-cache.json`
2. 使用者說「寫入」→ 直接讀取 `anime-cache.json` 並寫入目標檔案

> 若尚未執行「找動畫」（即 `anime-cache.json` 不存在），`write.js` 會直接回報錯誤，**不會詢問年份或季節**。

### 執行指令

```bash
node .cursor/skills/find-anime/write/write.js
```

不接受額外的年份、季節、評分參數，所有條件均來自 `anime-cache.json`。

### 觸發關鍵字

當使用者訊息中包含以下任一關鍵詞時，**直接讀取 `anime-cache.json` 並執行寫入**，不額外詢問：

- `寫入`
- `寫入清單`
- `寫入動畫`
- `把清單寫入`
- `存到檔案`

### 互動規則

偵測到「寫入」關鍵字後，**直接執行**，不詢問年份、季節或評分，因為資料已暫存在 `anime-cache.json` 中。

若使用者輸入如「寫入 2025 冬季動畫」這類帶有參數的句型，流程為：
1. 先觸發「找動畫」（`find.js 2025 winter`）獲取並暫存資料
2. 再呼叫 `write.js` 從暫存寫入

核心 `write.js` 永遠只讀 `anime-cache.json`，不直接接收年份參數。

### 批次內容格式

- **第一行（批次標題）**

  格式：`<year>  <季節(中文)>  >minScore    (更新時間: YYYY-MM-DD HH:mm)`

  季節中文對照：
  - `winter` → `冬`
  - `spring` → `春`
  - `summer` → `夏`
  - `fall`   → `秋`

- **後續每行（動畫資料列）**

  採用 Markdown 無序清單格式，`score` 以粗體凸顯，`episodes` 與 `type` 以括號包住作為補充資訊：

  ```markdown
  - 咒術回戰 懷玉·玉折 / 澀谷事變  **8.71**  (23 · TV)
  - 某某動畫  **-**  (未定 · 未知)
  ```

  - `name_cn`：優先使用 `item.name_cn`，若為空則退回其他標題欄位，或顯示 `未命名`
  - `score`：有值時顯示兩位小數（例如 `8.71`），無值時顯示 `-`，以粗體顯示
  - `episodes`：有值時直接顯示集數（例如 `23`），無值時顯示 `未定`
  - `type`：有值時顯示原始字串（例如 `TV`, `ONA`），無值時顯示 `未知`

- **分隔線**

  每一批最後會輸出一條分隔線：
  ```text
  ------------------------------
  ```

### 向上堆疊規則

1. 讀取 `D:/MCPfilesystem/動畫.md` 目前內容（若檔案不存在則當作空字串）
2. 產生新的批次文字 `batchText`
3. 新批次放在最前面：

   ```text
   batchText

   <舊內容>
   ```

4. 寫回同一個檔案，使「越新的批次越上面」

### 執行成功輸出範例

```
✅ 已寫入一批動畫資料到：D:/MCPfilesystem/動畫.md
   來源：2023 夏季，評分門檻 > 7.70，共 24 部
```

### 執行失敗輸出範例

```
❌ 寫入失敗：找不到 anime-cache.json，請先執行「找動畫」產生暫存（預期路徑：...）
```
