## 自動補齊缺失季度

### 專案位置

`d:\練習\spiderMAL`

### 功能說明

自動偵測 `D:/GoogleDrive_Sync/動畫.md` 內最新一季與「執行當下時間的上一季」之間的差距，分批呼叫 API 搜尋並寫入所有缺失的季度。

### 四季定義

| 季節 | 開播月份 | 對應英文 |
|------|---------|---------|
| 冬季 | 1、2、3 月 | `winter` |
| 春季 | 4、5、6 月 | `spring` |
| 夏季 | 7、8、9 月 | `summer` |
| 秋季 | 10、11、12 月 | `fall` |

### 「上一季」推算邏輯

根據執行當下的年月，先判斷當前所處的季節，再往回退一季：

| 當前月份 | 當前季節 | 上一季 |
|---------|---------|-------|
| 1-3 月  | 冬季    | 前一年 秋季 |
| 4-6 月  | 春季    | 同年 冬季 |
| 7-9 月  | 夏季    | 同年 春季 |
| 10-12 月 | 秋季   | 同年 夏季 |

### 執行指令

```bash
node .cursor/skills/find-anime/update/update.js
```

> 腳本與 Cursor / Claude 架構共用，位於 `.cursor/skills/find-anime/update/update.js`。
> 不接受任何參數，所有資訊自動推算。
> 評分門檻固定為 `7.70`。

### 觸發關鍵字

當使用者訊息中包含以下任一關鍵詞時，**直接執行**，不額外詢問：

- `更新`
- `更新動畫`
- `補齊`
- `同步`

### 完整流程

1. 取得執行當下的年月，推算「上一季」
2. 讀取 `D:/GoogleDrive_Sync/動畫.md` 前 5 行，解析出最新一季的年份與季節
3. 比對：
   - 若 **最新一季 === 上一季**（或更新）→ 輸出「已是最新，不需更新」並結束
   - 若 **最新一季 < 上一季** → 計算兩者之間所有缺失的季度
4. 從最舊的缺失季度開始，依序：
   - 呼叫 Jikan API 搜尋該季動畫
   - 篩選評分 >= 7.70 的動畫
   - 透過 Bangumi 取得中文名稱
   - 格式化後以「向上堆疊」方式寫入 `動畫.md`
5. 全部完成後輸出摘要

   篩選後：12 部（評分 >= 7.70）
   查詢 Bangumi 中文名稱...
   ✅ 2026 winter 已寫入（12 部）

──────────────────────────────────────────────────

✅ 全部完成！已補齊 1 個季度到：D:/GoogleDrive_Sync/動畫.md
```

```
